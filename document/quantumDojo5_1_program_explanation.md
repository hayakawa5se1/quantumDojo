対象ファイル: [`notebook/.ipynb_checkpoints/quantumDojo5_1-checkpoint.ipynb`](c:/Users/takat/Documents/program/qiskit/notebook/.ipynb_checkpoints/quantumDojo5_1-checkpoint.ipynb)

## 概要

このノートブックは、2量子ビット系のハミルトニアンに対して、変分量子固有値法 VQE の考え方を用いて基底エネルギーを求めるプログラムである。  
ただし、実際の量子計算機や Qiskit のシミュレータ backend を直接使っているのではなく、`numpy` による状態ベクトル計算で量子状態を追跡している点が特徴である。Qiskit は主に量子回路の可視化に用いられている。

## プログラム全体の流れ

1. パウリ行列と射影演算子を定義する
2. それらを2量子ビット系全体の演算子へ拡張する
3. 回転ゲートと CNOT ゲートを使った試行量子回路を定義する
4. 回路から得られる量子状態に対し、ハミルトニアンの期待値を計算する
5. 古典最適化により期待値が最小となるパラメータを探索する
6. 厳密対角化と比較して結果の妥当性を確認する
7. 最後に、ノイズを加えた場合の VQE のふるまいも確認する

## 各部分の構造と意味

### 1. パウリ行列と基本演算子の定義

最初のセルでは `pI`, `pX`, `pY`, `pZ` が定義されている。これらは1量子ビットに作用する基本演算子である。

- `pI`: 恒等演算子
- `pX`: パウリ X 行列
- `pY`: パウリ Y 行列
- `pZ`: パウリ Z 行列

物理学的には、量子ビットは二準位系であり、これらの行列はその状態変化や観測量を表す。特に `pZ` は計算基底 \(|0\rangle, |1\rangle\) と深く関係し、`pX` は基底状態同士を入れ替える働きを持つ。

また、`pP0`, `pP1` はそれぞれ \(|0\rangle\langle 0|\)、\(|1\rangle\langle 1|\) に対応する射影演算子であり、制御ゲートを行列表現で構成するために使われる。

### 2. 2量子ビット演算子への拡張

次のセルでは `X[i]`, `Y[i]`, `Z[i]`, `H[i]`, `P0[i]`, `P1[i]` がクロネッカー積 `np.kron` によって定義されている。

これは、1量子ビット演算子を2量子ビット全体のヒルベルト空間に埋め込む操作である。  
たとえば `X[0]` は「0番目の量子ビットにだけ X を作用させ、もう一方には何もしない演算子」を意味する。

2量子ビット系の状態空間は4次元なので、ここで得られる演算子は \(4 \times 4\) 行列になる。

### 3. 2量子ビットゲートの構成

`CX[i][j]` と `CZ[i][j]` は、射影演算子を使って制御ゲートを行列として構築している。

たとえば

```python
CX[i][j] = P0[i] + np.dot(P1[i], X[j])
```

は、「制御量子ビット `i` が \(|0\rangle\) なら何もしない」「\(|1\rangle\) なら標的量子ビット `j` に X を作用させる」という CNOT ゲートの定義そのものである。

物理学的には、このゲートにより量子もつれを作ることができる。VQE では、基底状態がもつれ状態を含むことが多いため、CNOT のような2量子ビットゲートが重要になる。

### 4. 回転ゲートの定義

`RX`, `RY`, `RZ` は

\[
R_X(\theta)=e^{-i\theta X/2}, \quad
R_Y(\theta)=e^{-i\theta Y/2}, \quad
R_Z(\theta)=e^{-i\theta Z/2}
\]

として定義されている。

これらはブロッホ球上で量子状態を各軸のまわりに回転させるユニタリ変換であり、量子回路中の連続的なパラメータ操作を担う。VQE では、この回転角が変分パラメータとして最適化される。

### 5. 初期状態の生成

`StateZeros(2)` は \(|00\rangle\) に対応する状態ベクトルを返す。

これは計算基底の最も単純な初期状態であり、ここから量子ゲートを順に作用させて試行状態を作る。

### 6. ハミルトニアンの定義

`M` は、複数のパウリ演算子の線形結合として定義されている。

```python
M = (-3.8505 * Ide - 0.2288 * X[1] - 1.0466 * Z[1] - ...
```

物理学的には、これは量子系のエネルギーを表すハミルトニアンである。  
VQE の目標は、このハミルトニアンの最小固有値、すなわち基底状態エネルギーを近似的に求めることにある。

### 7. 試行量子回路と試行状態

`TwoQubitPQC(phi)` は、6個のパラメータ `phi` を持つ変分量子回路を表している。

回路の流れは次の通りである。

1. \(|00\rangle\) を初期状態とする
2. 量子ビット0に `RX`, `RZ`
3. 量子ビット1に `RX`, `RZ`
4. `CX[1][0]` を適用
5. 量子ビット1にさらに `RZ`, `RX`

これにより、パラメータ `phi` に依存する試行状態 \(|\psi(\phi)\rangle\) が作られる。

物理学的には、この試行状態は「基底状態の候補」である。VQE では、真の基底状態を直接求めるのではなく、この候補の中で最も低エネルギーなものを探す。

### 8. 期待値の計算

`ExpectVal(Operator, State)` は

\[
\langle \psi | O | \psi \rangle
\]

を計算する関数である。

ここで `Operator` がハミルトニアン `M` なら、その状態における平均エネルギーを意味する。変分原理により、任意の試行状態に対するこの期待値は真の基底エネルギー以上になるため、これを最小化すれば基底状態に近づける。

### 9. 古典最適化

`cost(phi)` は

```python
ExpectVal(M, TwoQubitPQC(phi))
```

を返しており、これが VQE における目的関数である。

`scipy.optimize.minimize` がこの関数を最小化し、最適なパラメータ `res.x` を探す。  
つまり、

- 量子側: パラメータから状態を作り、エネルギー期待値を返す
- 古典側: その値を下げるようにパラメータを更新する

というハイブリッド構造になっている。

### 10. 回路図の生成

`build_two_qubit_pqc_circuit(phi)` は Qiskit の `QuantumCircuit` を用いて、同じゲート列を可視化用の回路として生成している。

この部分は物理計算の本体ではなく、最適化後に「どのような量子回路を用いたか」を図として確認するための補助的役割を持つ。

### 11. 厳密解との比較

`scipy.linalg.eigh(M)` により、ハミルトニアン `M` を厳密に対角化し、最小固有値を求めている。

これは小さい系だからできる方法であり、VQE の結果 `cost(res.x)` と比較することで、近似の精度を確認できる。

### 12. ノイズを入れた拡張

後半では

```python
ExpectVal(M,TwoQubitPQC(phi)) + np.random.normal(0,0.01)
```

としてノイズを加えている。

これは量子実機での測定誤差や有限ショット誤差を模擬したものであり、理想的なシミュレーションと実機に近い状況との違いを体験するための設定である。

## まとめ

このノートブックは、VQE の最小構成を自前の線形代数で実装した教材である。  
本質的な流れは、

1. 量子演算子を行列として定義する
2. 変分量子回路で試行状態を作る
3. ハミルトニアンの期待値を評価する
4. 古典最適化でその期待値を最小化する

というものである。

物理学的には、「量子系の基底状態を、変分原理を使って近似的に求める」という問題設定になっており、量子計算と古典計算の協調によってエネルギー最小化を行う点がこのプログラムの中心である。
