# quantumDojo5_1 量子端子・量子回路要素の整理表

対象ファイル: [`notebook/.ipynb_checkpoints/quantumDojo5_1-checkpoint.ipynb`](c:/Users/takat/Documents/program/qiskit/notebook/.ipynb_checkpoints/quantumDojo5_1-checkpoint.ipynb)

このファイルで量子回路の生成に直接関わっている要素について、Qiskit での記載方法、物理学的意味、量子回路での記号に注目して整理する。

| 量子端子・要素 | Qiskitでの記載方法 | 物理学的な意味 | 量子回路での記号 |
|---|---|---|---|
| 量子ビット | `qc = QuantumCircuit(2)` | 2つの量子ビットからなる系を定義する。各量子ビットは \(|0\rangle\) と \(|1\rangle\) の重ね合わせを取れる二準位系である。 | `q0: ---`, `q1: ---` |
| 初期状態 | Qiskit では回路作成時に暗黙に \(|00\rangle\) から開始 | 回路の出発点となる基準状態。ノートブック内の自作シミュレーションでは `StateZeros(2)` で明示的に作られている。 | 回路左端の `|0⟩` |
| X軸回転ゲート | `qc.rx(phi[0], 0)` など | ブロッホ球で量子状態を X 軸まわりに回転させる。重ね合わせ状態を作る基本操作。数式では \(R_X(\phi)=e^{-i\phi X/2}\)。 | `Rx(φ)` |
| Z軸回転ゲート | `qc.rz(phi[1], 0)` など | ブロッホ球で量子状態を Z 軸まわりに回転させる。主に相対位相を変化させる操作。数式では \(R_Z(\phi)=e^{-i\phi Z/2}\)。 | `Rz(φ)` |
| CNOTゲート | `qc.cx(1, 0)` | 制御量子ビットが \(|1\rangle\) のときだけ標的量子ビットに X を作用させる2量子ビットゲート。量子もつれを作る中心的な操作。 | 制御側 `●`、標的側 `⊕` |
| 変分パラメータ | `phi[0]` から `phi[5]` | 回転角を表すパラメータ。古典最適化によって更新され、エネルギー期待値を下げる方向に調整される。 | `Rx(φ0)`, `Rz(φ1)` など |
| 試行量子回路 | `build_two_qubit_pqc_circuit(phi)` | 基底状態候補を生成する回路。回転ゲートともつれ生成ゲートを組み合わせて、多様な量子状態を表現できるようにする。 | 回路全体として表現 |

## このノートブックで生成される回路の概形

```text
q0: ─ Rx(φ0) ─ Rz(φ1) ─────X────────────
                               │
q1: ─ Rx(φ2) ─ Rz(φ3) ──●── Rz(φ4) ─ Rx(φ5)
```

## 補足

このノートブックでは `X`, `Y`, `Z`, `P0`, `P1`, `CX` なども `numpy` 行列として定義されている。  
ただし、これらは回路図にそのまま箱として並べるためというより、量子ゲートやハミルトニアンを線形代数的に構成するための演算子である。

一方、Qiskit で回路図として直接記述されている量子回路要素は、主に次の3種類である。

- `QuantumCircuit`
- `rx`
- `rz`
- `cx`

したがって、このファイルにおける量子回路生成の中心は「単一量子ビット回転」と「CNOT によるもつれ生成」であるといえる。
